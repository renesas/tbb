🚀 Welcome to the TBB Repository!

🧩 What is a TBB?​

A TBB (Technology Building Block) is a curated bundle of technical collateral crafted to help customers accelerate solution development using Renesas MCUs.

Address a market or segment

TBBs captivate a solution designed to be (for own project ... ) , reusable project assets, and solution blueprints that reduce development friction and improve time-to-market.

🎯 Objectives of a TBB​

  • Expand Reach — Bring Renesas technology to new global markets.

  • Stand Out — Offer differentiated and relevant technical solutions.

  • Build Loyalty — Deliver value and maintain customer engagement.

  • Empower Teams — Give engineering a path to impact product and business outcomes.

📦 What's Inside a TBB?​

Each TBB may include:

  • ✅ Proof of Concept (PoC)

  • 🧑‍💻 Sample Projects

  • 🛠️ Hardware Design Files / PCBs

  • 🧠 Training Content

  • 📚 User Manuals

  • 🗃️ Application Notes, Lab Guides, Slide Decks

🧭 TBB Lifecycle Flow​

Here's a visual breakdown of how a TBB is created, from request to public release:

graph TD
  A[TBB Request (OFJ)] --> B[AE Mgmt Review & Assignment]
  B --> C{TBB Author Assigned?}
  C -->|Yes| D[Create APPENG JIRA Ticket]

🔁 TBB Process Summary​

🔹 1. Request Phase​

Submitted via OFJ from roles like:

  • Sales

  • FAEs

  • BD

  • PMM

  • Customers

Mandatory Info:

  • Customer Name

  • Device & Hardware (EVK, FPB, custom board)

  • Feature focus

  • Desired collateral

  • Business case justification

🔹 2. Triage & Assignment​

  • AE Managers review request

  • Assign to appropriate Application Engineer

🔹 3. Development​

  • Create APPENG JIRA Task (Label: TBB)

  • Build project in TBB Private GitHub Repo

  • Document deliverables and track progress using Smart Checklists, subtasks, and comments

🔹 4. Release​

  • Submit Pull Request into main

  • Auto-mirror to Public Repo

  • External contributions (fixes, updates) accepted via PRs

👷 Author Instructions​

✅ One-time Setup​

  • GitHub: Use your Renesas email to join the Renesas GitHub Org

  • Access:

  • 📁 Private GitHub Repository

  • 🗂️ APPENG JIRA

✅ Per-TBB Actions​

  1. Create a JIRA Ticket

  • Use APPENG Project: jira.eng.renesas.com

  • Title format: TBB – <Project Name>

  • Use TBB label

  1. Develop Content

  • Follow private repo folder structure: <TBB_NAME>/PROJECT/, /COLLATERAL/

  • Use feature branches or forking

  1. Release

  • Submit Pull Request → Triggers automated publish to public

  • Public repo mirrors all content

  • External PRs are allowed and managed by regional GitHub admins

🧠 Tips & Tools​

  • ✅ Use Smart Checklists for project tasks

  • 🧩 Track all activity via JIRA Comments

  • 🔄 Maintain version control via GitHub
